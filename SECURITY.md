# Security

This Security outlines the measures and procedures we take to ensure the security of our ReApptor projects.

## Reporting Security Issues or Vulnerabilities

If you believe you have found a security vulnerability in any ReApptor-owned repository, please report it to us through coordinated disclosure.

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please send an email to contact[@]reapptor.com.

Please include as much of the information listed below as you can to help us better understand and resolve the issue:

  * The type of issue (buffer overflow, SQL injection, or cross-site scripting, etc...);
  * Full paths of source file(s) related to the manifestation of the issue;
  * The location of the affected source code (tag/branch/commit or direct URL);
  * Any special configuration required to reproduce the issue (if possible);
  * Step-by-step instructions how to reproduce the issue;
  * Proof-of-concept or exploit code (if possible);
  * Assumed impact of the issue, including how an attacker might exploit the issue (if possible);

This information will help us fix the issue asap!
